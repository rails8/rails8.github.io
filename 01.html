We start with a brand new Rails app and give it a minimal setup:

  1. ▶︎ Create a Rails repo calling rails new marketplace with a few options: --main to point to the most recent version of Rails, -d postgresql to use PostgreSQL as the database, and --skip-solid to skip Solid (spoiler: you’ll use Redis instead).
  2. ▶︎ Clean up the Gemfile to only include required libraries at their most recent version.
  3. ▶︎ Replace ' 3000' with ' 2026' in the code so the server will run locally at a fancy URL. ✨
  4. ▶︎ Tell Rails not to generate unit tests for new models (spoiler: you’ll use system tests instead).
  5. ▶︎ Create the local database with bin/rails db:create.
  6. ▶︎ Start the server and open http://localhost:2026 to confirm everything is set up correctly.
